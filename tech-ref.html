The world's largest cloud platform. Every piece of Command Legacy runs here. No physical servers, no data centers to manage.

Data Storage
DynamoDB
›
What it is
AWS's enterprise key-value database. Stores all officer records, tiers, and access credentials — encrypted, fast, no third-party access. Millisecond lookups at any scale.
Say it like this
"Officer data is stored in Amazon DynamoDB — an enterprise-grade encrypted database that meets federal security requirements. No officer data touches any third-party system."
CEO note
Ask Tim: as enterprise contracts grow, how are we handling usage reporting? DynamoDB doesn't natively support complex analytics — we'll need a plan.
⚡
Access Engine
Lambda + API Gateway
›
What it is
Every app launch triggers Lambda, which checks DynamoDB and issues a signed digital badge (JWT). API Gateway is the secure front door. Zero third-party tools in this pipeline.
Say it like this
"Access is verified on every app launch through our serverless entitlement engine. No officer can access the platform without passing our automated gate — regardless of credentials."
CEO note
Ask Tim: what happens if a token expires mid-shift in a dead zone? Fail open (stays in) or fail closed (locked out)? Law enforcement needs a clear answer before enterprise pitches.
🔐
Encryption
AWS KMS + Secrets Manager
›
What it is
All encryption keys and credentials stored in AWS Key Management Service. Never in code, never in config files, never in logs. 256-bit encryption at rest and in transit.
Say it like this
"Every credential and encryption key is stored in AWS KMS — the same service used by financial institutions and federal agencies. Nothing sensitive ever touches our application code."

AI Legal Research
Sherlock AI
›
What it is
Officers ask plain-language questions about statutes. Sherlock returns accurate, sourced answers. Temperature set to 0.2 to maximize factual accuracy and minimize hallucination.
Say it like this
"Sherlock gives officers instant access to legal statutes in plain English — no law degree required. Citations are always provided so officers can verify every answer."
When asked "which AI?"
"OpenAI via secure API — officer query data is not used for training." Never say "we use OpenAI" without that context.

Community
Roll Call Room
›
What it is
Private, verified peer community for law enforcement supervisors only. Every member individually vetted. Runs on Discourse. The verified community is the strategic moat.
Say it like this
"Roll Call Room is a private professional community — think LinkedIn but exclusively for verified law enforcement. Every member has been individually vetted."
Strategic note
A competitor can copy Sherlock's features in months. They cannot replicate 300 verified supervisors who trust each other. The community is the moat.

Tough Questions
Q
Where does our officers' data live?
"Entirely within Amazon Web Services US-East data centers. It never touches a third-party system. We can provide a full data flow diagram for your IT security team."
Q
Is this CJIS compliant?
"Our architecture was designed to meet CJIS security policy requirements from day one. We're happy to walk your IT security director through our technical documentation."
Q
What happens if there's a data breach?
"We have point-in-time recovery on all databases, CloudWatch monitoring across the entire stack, and an incident response process. We can share our security runbook."
Q
Who else has access to our data?
"Only authorized Command Legacy personnel with role-based access controls. No vendors, no third parties, no AI training on your data. We can provide a full access control policy."
Q
Do you support MFA or SSO?
"Our authentication architecture is intentionally lean for the controlled launch phase. MFA and agency SSO are on our near-term roadmap as we scale to larger departments."
Q
Which AI model powers Sherlock?
"OpenAI via secure API. Officer query data is never used for model training. The data boundary is structural — enforced at the API level, not just a policy."
